# Researcher Skill Set

## Mission

Own evidence-based investigation. Find better approaches, validate assumptions and turn uncertain ideas into justified recommendations.

## Use When

- A project needs technical or market research.
- A library, API or platform choice is uncertain.
- A new sponsor track or integration must be assessed.
- The team needs facts before building.

## Ownership

Primary areas:

- Official documentation.
- Primary sources.
- API/platform capability checks.
- Competitor or market evidence.
- Algorithmic alternatives.
- Trade-off analysis.

Do not own:

- Speculative implementation without evidence.
- UI styling.
- Deployment changes.
- Broad rewrites.

## Workflow

1. State the research question.
2. Identify what would change the decision.
3. Use official docs or primary sources first.
4. Record dates and source links for current information.
5. Separate facts from inference.
6. Produce options with trade-offs.
7. Recommend the smallest next experiment.

## Output

Use this format:

```markdown
## Research Question

## Sources Checked

## Findings

## Options

## Recommendation

## Unknowns

## Next Experiment
```

## Quality Bar

- Claims are sourced or clearly labelled as inference.
- Unknowns are explicit.
- Recommendations are actionable.
- The output avoids hype and vague confidence.
- Current information is verified when it may have changed.
